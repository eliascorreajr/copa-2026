# Mata-Mata Copa 2026 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add automatic group standings, best third-place ranking, knockout bracket generation, winner propagation, and manual correction support without breaking the existing bolao flows.

**Architecture:** Keep current HTML/CSS/vanilla JS/Firebase architecture. Add pure JavaScript modules for World Cup competition logic, test them with Node, then integrate them into `admin.html` and Firestore as AdminSuper-only operations. Keep bolao scoring (`+7/+3/-1`) separate from World Cup standings (`3/1/0`).

**Tech Stack:** HTML, CSS, JavaScript ES modules, Firebase Auth, Cloud Firestore, GitHub Pages, Node syntax/smoke tests.

## 1. Diagnostico Tecnico

### Projeto atual

- `data/matches.json` e a tabela base: 104 jogos, 72 de grupos e 32 de mata-mata.
- `js/scoring.js` e a logica de pontuacao do bolao. Deve continuar exclusiva para palpites.
- `js/matches.js` carrega jogos, calcula dia, trava por dia e informacoes de lock.
- `js/sofascore.js` sincroniza resultados, mas nao deve ser usado como fonte ativa.
- `admin.html` concentra painel administrativo, resultado manual, travas, usuarios, exportacao e logs.
- `firestore.rules` ja protege `users`, `guesses`, `matches`, `locks`, `matchLocks`, `logs`.

### Onde estao os resultados oficiais

Os resultados oficiais usados pelo app ficam em `matches/{matchId}` no Firestore. O documento local `data/matches.json` contem a tabela, mas nao deve ser tratado como fonte de resultado final.

### Como fases, rodadas e grupos sao identificados

- Fase de grupos: `groupMatches[]`, com `group`, `round`, `id`, `homeTeam`, `awayTeam`, `date`.
- Mata-mata: `knockoutMatches[]`, com `stage`, `id`, `homeTeam`, `awayTeam`, `date`.
- Resultados no Firestore usam `matchId` numerico e documento `match_{matchId}` quando casados com a tabela local.

### Como adicionar mata-mata sem quebrar grupos

- Nao alterar a estrutura de palpites existentes.
- Nao alterar `js/scoring.js` exceto se for necessario para exportar helper ja existente.
- Adicionar modulos novos para classificacao e bracket.
- Fazer o painel admin gerar snapshots em novas collections.
- Atualizar `data/matches.json` ou renderizacao do mata-mata apenas quando confrontos estiverem definidos e revisados.

## 6. Regras de Calculo em Pseudocodigo

### Classificacao de grupos

```text
para cada grupo:
  criar stats zeradas para cada selecao do grupo
  para cada jogo do grupo:
    se resultado incompleto:
      marcar grupo como incompleto
      continuar
    atualizar jogos disputados, gols pro, gols contra
    se homeScore > awayScore:
      mandante ganha 3, visitante perde
    se homeScore < awayScore:
      visitante ganha 3, mandante perde
    se empate:
      ambos ganham 1 ponto e 1 empate
  ordenar por pontos
  para blocos empatados:
    aplicar confronto direto entre empatados
    aplicar saldo de confronto direto
    aplicar gols pro de confronto direto
    reaplicar se o bloco reduzir
    aplicar saldo geral
    aplicar gols pro geral
    se ainda empatar:
      marcar needsManualTiebreak
  aplicar overrides manuais de posicao
  retornar snapshot do grupo
```

### Melhores terceiros

```text
terceiros = pegar posicao 3 de cada grupo
se algum grupo incompleto:
  ranking completo = false
ordenar terceiros por:
  pontos desc
  saldo desc
  gols pro desc
se ainda empatar:
  marcar needsManualTiebreak
marcar os 8 primeiros como qualified = true quando ranking estiver resolvido
aplicar overrides manuais
```

### Bracket da fase de 32

```text
carregar template oficial M73-M88
resolver sources fixas:
  1A = grupo A posicao 1
  2B = grupo B posicao 2
resolver slots de terceiro:
  obter grupos dos 8 terceiros classificados
  buscar combinacao oficial na tabela de alocacao
  se combinacao nao existir:
    deixar slot pendente
para cada jogo:
  se dois lados resolvidos: status = defined
  se um lado resolvido: status = partially_defined
  se nenhum lado resolvido: status = pending
preservar campos com manualOverride
```

### Propagacao de vencedores

```text
para cada jogo eliminatorio finalizado:
  determinar vencedor:
    se homeScore != awayScore: maior placar vence
    se homeScore == awayScore: exigir winner manual
  escrever winner e loser no bracketMatches/{matchId}
  localizar destino:
    W73 -> M90 slot home/away conforme template
    L101 -> M103
  se destino nao tem override manual no slot:
    preencher team no slot de destino
  recalcular status do destino
```
